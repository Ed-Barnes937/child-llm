# Outstanding Process Steps

Steps in the product development process to address at the right time. Not blocking implementation.

## Before/During Build
- **Testing / Red Team Strategy** — plan for testing guardrails, prompt injection resistance, output filtering. Child safety is the product — can't be an afterthought.
- **Data Model / Schema Design** — Postgres tables for accounts, presets, conversations, messages, flags, device tokens. Bridge between architecture and implementation.
- **Content Strategy** — onboarding calibration questions, preset definitions, system prompt templates, "Inspire me" topics, intent categories, error messages.

## Before Launch
- **User Validation** — show the MVP to 5-10 parents. "Would you pay for this? What's missing? What worries you?" Cheapest way to find wrong assumptions.
- **Success Metrics / KPIs** — define what to measure and instrument V1 accordingly (conversion, retention, engagement, flag accuracy).
- **Go-to-Market Strategy** — how parents find the product. Can develop in parallel with later implementation.
- **Naming / Branding** — product name, domain, app identity. Blocks marketing and app store listing.
- **Design Pass** — custom theme over Shadcn defaults. Warm, trustworthy, child-friendly. Not clinical SaaS.
